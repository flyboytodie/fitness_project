# 健身记录应用 - 数据统计与显示优化建议

---

## 一、数据统计增强

### 1.1 训练量趋势图

**功能描述**：
- 显示每周训练总量（组数 × 次数）的趋势变化
- 使用折线图展示近8周的数据
- 标注每周训练天数

**实现建议**：
- 使用 Chart.js 绘制折线图
- 数据来源：`appData.workouts`
- 计算方式：每周所有训练的 `sets × avg(reps)`

**优先级**：⭐⭐⭐ 高

---

### 1.2 部位分布饼图

**功能描述**：
- 显示各肌群训练占比
- 使用饼图或环形图展示
- 悬停显示具体数值

**实现建议**：
- 使用 Chart.js 绘制饼图
- 统计最近30天的训练部位分布
- 支持按时间范围筛选

**优先级**：⭐⭐⭐ 高

---

### 1.3 连续训练天数 (Streak)

**功能描述**：
- 显示当前连续训练天数
- 显示历史最长连续天数
- 激励用户保持训练习惯

**实现建议**：
- 从 `appData.workouts` 计算连续日期
- 在首页显著位置显示
- 添加火焰图标增强视觉效果

**优先级**：⭐⭐ 中

---

### 1.4 个人纪录 (PR) 追踪

**功能描述**：
- 自动追踪每个动作的最佳成绩
- 记录最高重量、最多次数、最大容量
- 显示突破记录的日期

**实现建议**：
- 创建 `Storage.savePR()` 和 `Storage.getPRs()` 方法
- 在训练页面实时检测 PR 突破
- 显示 PR 徽章提示

**优先级**：⭐⭐⭐ 高

---

### 1.5 肌肉恢复状态

**功能描述**：
- 根据上次训练时间显示各部位恢复程度
- 使用颜色编码（绿色=已恢复，黄色=恢复中，红色=疲劳）
- 训练时推荐优先训练已恢复部位

**实现建议**：
- 计算每个部位上次训练距今天数
- 恢复周期：大肌群 48-72 小时，小肌群 24-48 小时
- 在训练页面显示恢复状态提示

**优先级**：⭐⭐ 中

---

### 1.6 周/月统计卡片

**功能描述**：
- 本周训练数据 vs 上周对比
- 显示环比增长率
- 热量消耗估算

**实现建议**：
- 计算每周训练组数、次数、动作数
- 计算环比增长百分比
- 使用箭头图标显示增减趋势

**优先级**：⭐⭐ 中

---

## 二、数据显示优化

### 2.1 首页数据看板

**功能描述**：
- 整合关键数据指标
- 包含：训练天数、连续天数、本周进度、热量消耗
- 卡片式布局，一目了然

**实现建议**：
- 四个象限展示核心指标
- 使用图标增强视觉效果
- 点击可跳转到对应页面

**优先级**：⭐⭐⭐ 高

---

### 2.2 动作进步追踪

**功能描述**：
- 同一动作不同日期的重量/次数对比
- 显示进步曲线
- 标注突破点

**实现建议**：
- 在训练记录详情页添加图表
- 使用折线图展示历史数据
- 高亮显示 PR 点

**优先级**：⭐⭐⭐ 高

---

### 2.3 进度条可视化

**功能描述**：
- 每周训练目标完成度进度条
- 热量摄入目标进度
- 身体指标目标进度

**实现建议**：
- 在首页和统计页面显示
- 使用渐变色进度条
- 支持自定义目标

**优先级**：⭐⭐ 中

---

### 2.4 里程碑徽章

**功能描述**：
- 完成特定成就解锁徽章
- 如：连续7天训练、完成100次训练、突破第一个 PR
- 在个人主页展示徽章墙

**实现建议**：
- 创建徽章数据结构
- 训练时检测成就解锁
- 使用 SVG 图标展示徽章

**优先级**：⭐ 低

---

### 2.5 数据对比视图

**功能描述**：
- 本月 vs 上月数据对比
- 去年同期对比（周年回顾）
- 生成对比报告

**实现建议**：
- 添加时间范围选择器
- 使用双轴图表对比数据
- 计算同比/环比增长率

**优先级**：⭐⭐ 中

## 五、技术实现建议

### 5.1 图表库选择

- **Chart.js**：轻量级，已在项目中使用
- **ECharts**：功能强大，支持更多图表类型
- **自定义 SVG**：高度定制化，适合简单图表

### 5.2 数据存储

- 使用 LocalStorage 存储 PR 数据
- 创建新的存储键：`fitness_prs`
- 数据结构：`{ exerciseName: { weight: number, reps: number, date: string } }`

### 5.3 性能优化

- 使用节流/防抖优化图表渲染
- 缓存计算结果避免重复计算
- 使用 requestAnimationFrame 优化动画
