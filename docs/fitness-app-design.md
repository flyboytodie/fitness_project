## 2. 删除确认与恢复机制

### 2.1. 功能设计

- **删除确认**：删除任何记录（训练、饮食、身体数据、模板）前，弹出自定义确认对话框
- **撤销功能**：删除后显示Toast提示，5秒内可点击"撤销"恢复数据
- **数据暂存**：删除的数据临时存储在内存中，超过撤销时限后自动清除

### 2.2. 技术实现

```javascript
// 删除数据暂存对象
const deletedItems = {
  workout: null,
  diet: null,
  body: null,
  template: null
};

// 撤销删除函数
function undoDelete(type) {
  const item = deletedItems[type];
  if (!item) return;
  
  // 根据类型恢复数据到对应数组
  if (type === 'workout') {
    const existingIndex = appData.workouts.findIndex(w => w.date === item.date);
    if (existingIndex >= 0) {
      appData.workouts[existingIndex] = item;
    } else {
      appData.workouts.push(item);
    }
  }
  // ...其他类型类似
  
  Storage.save(appData);
  deletedItems[type] = null;
  showToast('已恢复数据', null, null, 2000);
  navigateTo(currentPage);
}
```

## 3. 动作选择交互优化

### 3.1. 当前问题分析

| 问题 | 描述 | 影响 |
| :--- | :--- | :--- |
| 下拉框交互 | 使用`<select>`下拉框，用户体验差 | 用户需要多次点击才能选择动作 |
| 筛选不生效 | `filterExercises()`只更新变量，不重新渲染 | 搜索和筛选功能无效 |
| 移动端适配 | 下拉框在手机上显示不全 | 影响移动端使用体验 |

### 3.2. 交互设计

**方案：搜索 + 弹出列表**

### 3.3. 数据来源与动作库管理

#### 3.3.1. 动作来源

| 来源类型 | 说明 | 优先级 |
| :--- | :--- | :--- |
| **预设动作库** | 内置的标准动作（约20个） | 高（收藏优先） |
| **自定义动作** | 用户手动输入的新动作 | 中 |
| **历史记录** | 用户之前训练过的动作 | 低 |

| 功能 | 描述 |

| 删除动作 | 从动作库中移除不需要的动作 |
| 编辑动作 | 修改动作名称和分类 |
| 分类管理 | 支持自定义动作分类 |
| 收藏功能 | 标记常用动作，优先显示 |

**动作数据结构**：

interface Exercise {
  id: string;             // 唯一标识

#### 3.3.3. 自定义动作自动加入库

当用户在训练记录中输入一个不在动作库中的动作名称时：

1. 系统自动检测该动作是否存在
2. 若不存在，弹出提示询问是否添加到动作库
3. 用户确认后，自动添加到"自定义"分类
4. 下次训练时可直接选择该动作

### 3.4. 自动填充逻辑

选择动作后，自动填入最近一次训练的重量和次数数据：

```javascript
function onExerciseSelect(index, exerciseName) {
  const lastEx = getLastExerciseData(exerciseName);
  if (lastEx) {
    document.querySelector(`[name="weight-${index}"]`).value = lastEx.weight;
    document.querySelector(`[name="reps-${index}"]`).value = lastEx.sets.join('/');
  }
}
```

## 5. 饮食数据导入功能

### 5.1. 统一数据导入格式

支持通过**单个JSON文件**导入健身数据和饮食数据。

#### 5.1.1. JSON格式（推荐）

**文件格式**：`fitness_data.json`

```json
{
  "workouts": [
    {
      "date": "2026-05-01",
      "parts": ["胸", "三头"],
      "exercises": [
        {
          "name": "杠铃卧推",
          "weight": 60,
          "sets": [8, 8, 8, 6],
          "totalReps": 30,
          "notes": "加重5kg"
        },
        {
          "name": "哑铃飞鸟",
          "weight": 15,
          "sets": [12, 12, 10],
          "totalReps": 34,
          "notes": ""
        }
      ],
      "sets": 7
    }
  ],
  "diet": [
    {
      "date": "2026-05-01",
      "protein": 150,
      "carbs": 200,
      "fat": 60,
      "calories": 1800,
      "notes": "训练日"
    },
    {
      "date": "2026-05-02",
      "protein": 140,
      "carbs": 180,
      "fat": 55,
      "calories": 1650,
      "notes": "休息日"
    }
  ],
  "exerciseLibrary": [
    {
      "id": "custom-001",
      "name": "我的自定义动作",
      "category": "custom",
      "muscle": "其他",
      "starred": false,
      "createdAt": "2026-05-01T10:00:00Z",
      "updatedAt": "2026-05-01T10:00:00Z"
    }
  ]
}
```

**字段说明**：

| 顶层字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| workouts | array | 否 | 训练记录数组 |
| diet | array | 否 | 饮食记录数组 |
| exerciseLibrary | array | 否 | 动作库数据 |

**训练记录字段**：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| date | string | 是 | 日期，格式：YYYY-MM-DD |
| parts | array | 是 | 训练部位列表 |
| exercises | array | 是 | 动作列表 |
| sets | number | 是 | 总组数 |

**动作字段**：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| name | string | 是 | 动作名称 |
| weight | number | 是 | 重量（kg） |
| sets | array | 是 | 每组次数数组 |
| totalReps | number | 是 | 总次数 |
| notes | string | 否 | 备注 |

**饮食记录字段**：

| 字段 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| date | string | 是 | 日期，格式：YYYY-MM-DD |
| protein | number | 是 | 蛋白质（克） |
| carbs | number | 是 | 碳水化合物（克） |
| fat | number | 是 | 脂肪（克） |
| calories | number | 否 | 总热量，自动计算时可不填 |
| notes | string | 否 | 备注信息 |

### 5.2. 导入流程

```
选择文件 → 检测格式 → 解析文件 → 数据验证 → 显示预览 → 确认导入 → 保存数据
```

**格式检测**：
- 自动识别JSON/CSV/MD格式
- 根据文件内容智能判断

**验证规则**：

| 规则 | 说明 |
| :--- | :--- |
| 日期格式 | 必须为 YYYY-MM-DD |
| 数值范围 | 蛋白质/碳水/脂肪/重量 ≥ 0 |
| 热量计算 | calories = protein×4 + carbs×4 + fat×9 |
| 重复日期 | 导入时提示覆盖、跳过或合并 |
| 动作验证 | 训练记录中的动作名称不能为空 |
| 组数验证 | 每组次数必须为正整数 |

**导入预览**：

  sets: number[];            // 每组次数
