# 健身数据追踪应用 - 设计规格

## 1. Concept & Vision

一款极简主义的个人健身数据追踪应用，采用单文件HTML架构，双击浏览器即可使用。采用简洁数据风格（浅色背景+清晰数字展示），让用户专注于数据本身。核心价值：**健身前快速记录，健身后随时复盘**。

## 2. Design Language

### 美学方向
- **参考风格**：Notion + Apple Health 的简洁数据风
- **核心原则**：信息优先，减少视觉干扰

### 调色板
```
Primary:     #2563EB (蓝色主色调 - 活力健康)
Secondary:   #10B981 (绿色 - 进步/达成)
Accent:      #F59E0B (橙色 - 热量/警告)
Danger:      #EF4444 (红色 - 超出目标)
Background:  #F8FAFC (浅灰白背景)
Surface:     #FFFFFF (卡片白色)
Text:        #1E293B (深色文字)
Text-muted:  #64748B (次要文字)
Border:      #E2E8F0 (边框)
```

### 字体
- **主字体**：-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif
- **数据字体**："SF Mono", "Fira Code", monospace（用于数字展示）
- **字号**：基础 16px，标题 20-28px，数据展示 24-48px

### 间距系统
- 基础单位：4px
- 卡片间距：16px
- 页面边距：16px（移动端）
- 卡片圆角：12px

### 动效哲学
- 过渡：200ms ease-out
- 页面切换：淡入淡出
- 数据更新：轻微弹跳反馈
- 进度条：流畅填充动画

## 3. Layout & Structure

### 整体布局
- **移动优先**：单栏布局，最大宽度 480px 居中
- **底部导航**：5个核心Tab（首页、训练、饮食、身体、数据）
- **卡片式内容**：白色卡片承载数据，分组清晰

### 页面结构
```
┌─────────────────────────┐
│  Header (当前页面标题)   │
├─────────────────────────┤
│                         │
│     Main Content        │
│     (可滚动区域)         │
│                         │
├─────────────────────────┤
│  Bottom Navigation       │
│  [首页][训练][饮食][身体][数据]
└─────────────────────────┘
```

### 响应式策略
- **< 640px**：单栏，底部导航
- **≥ 640px**：双栏布局，侧边导航

## 4. Features & Interactions

### 4.1 首页（仪表盘）
**功能**：展示核心数据和快速入口

**内容**：
- 今日概览卡片（训练状态、摄入热量、体重）
- 本周训练统计（次数、时长、总组数）
- 最近进步亮点（自动检测新记录的动作进步）
- 快速操作按钮（开始训练、记录饮食、记录体重）

**交互**：
- 下拉刷新数据
- 点击卡片进入详情

### 4.2 训练记录
**功能**：记录每次训练

**训练录入流程**：
```
1. 选择训练日期（默认今天）
2. 选择目标部位（可多选）
   - 胸+三头、背+二头、肩、腿、全身
3. 添加动作
   - 从历史动作选择 OR 新增动作
   - 输入：动作名称、重量、每组次数
   - 格式：55kg × (8/8/8/8/8) — 每组次数可不同
   - 可选：组数、备注
4. 保存训练
```

**动作记录格式**（参考用户习惯）：
```javascript
{
  name: "杠铃卧推",
  weight: 55,        // kg
  weightUnit: "kg",
  sets: [
    { reps: 8 },
    { reps: 8 },
    { reps: 8 },
    { reps: 8 },
    { reps: 8 }
  ],
  totalReps: 40,
  notes: "最佳表现！5组全稳定"
}
```

**训练历史**：
- 按日期倒序展示
- 支持按部位筛选
- 点击展开查看详情

### 4.3 饮食记录
**功能**：记录每日营养摄入

**录入格式**（参考用户习惯）：
```javascript
{
  date: "2026-05-07",
  protein: 136,      // g
  carbs: 162,        // g
  fat: 34,           // g
  calories: 1461,    // kcal
  meals: {
    breakfast: { protein: 50, carbs: 100, fat: 10, calories: 259 },
    lunch: { protein: 30, carbs: 50, fat: 20, calories: 340 },
    afternoon: { protein: 40, carbs: 20, fat: 5, calories: 510 },
    postWorkout: { protein: 16, carbs: 52, fat: 9, calories: 352 }
  },
  notes: "早餐259/午餐340/下午510/练后352"
}
```

**热量目标设置**：
- 可设置每日目标（默认 2000 kcal）
- 显示摄入/目标对比条

### 4.4 身体数据
**功能**：记录体重和身体围度

**录入格式**：
```javascript
{
  date: "2026-05-07",
  weight: 75.5,      // kg
  waist: 80,         // cm
  notes: "空腹测量"
}
```

**数据展示**：
- 最新数值卡片
- 趋势图表（折线图）

### 4.5 数据分析
**功能**：全面的数据统计和可视化

**子页面**：
1. **进步追踪**
   - 各动作重量/次数历史曲线
   - 自动标注进步节点（如"首次突破"、"加重成功"）

2. **训练统计**
   - 总训练次数
   - 各部位训练频率
   - 平均每周训练次数
   - 最长连续训练天数

3. **热量分析**
   - 每日摄入热量趋势
   - 蛋白质/碳水/脂肪分布

4. **身体变化**
   - 体重趋势图
   - 腰围趋势图

### 4.6 数据管理
**功能**：导入/导出/备份

- **导出JSON**：下载所有数据为JSON文件
- **导入JSON**：从备份恢复数据
- **清空数据**：确认后清除所有记录

## 6. Technical Approach

### 架构
- **单文件HTML**：所有CSS和JavaScript内联
- **无依赖**：纯原生JS，不使用框架
- **本地存储**：localStorage持久化数据

### 数据模型
```javascript
// 应用状态
{
  settings: {
    calorieGoal: 2000,
    weightUnit: "kg"
  },
  workouts: [...],      // 训练记录数组
  diet: [...],          // 饮食记录数组
  body: [...],          // 身体数据数组
  exerciseLibrary: [...] // 动作库（自动积累）
}
```

### 数据持久化
- localStorage.setItem('fitness_data', JSON.stringify(data))
- 每次操作后自动保存
- 页面加载时读取

### 图表实现
- 使用 Chart.js CDN（轻量、支持本地）
- 或使用纯Canvas自定义绘制

### 响应式实现
- CSS媒体查询
- Flexbox布局
- viewport meta标签

### 性能优化
- 防抖处理输入
- 懒加载图表
- 最小化重绘

## 7. 预设动作库

基于用户习惯，预设常用动作：
```
胸部：杠铃卧推、上斜哑铃卧推、固定器械推胸、固定器械夹胸、龙门架夹胸
背部：引体向上、器械拉背、高位下拉、器械划船、龙门架V把划船
肩部：哑铃推肩、哑铃飞鸟、实力推、后束飞鸟、杠铃提拉
腿：深蹲、腿弯举、器械内收
三头：绳索下压
二头：哑铃弯举
有氧：跑步、骑行、游泳
```

## 8. 进步检测规则

自动检测并标注进步：
- **重量增加**：同一动作使用更重的重量
- **次数增加**：同一重量下完成更多次数
- **稳定性提升**：之前需要降重的动作现在全程不降重
- **首次突破**：某动作首次达到特定里程碑
